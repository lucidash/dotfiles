# 세션 리뷰 및 지식 베이스 개선

현재 세션에서 학습한 내용을 검토하고, CLAUDE.md 또는 skills(commands)에 반영할 개선사항이 있는지 확인합니다.

## 목적
- 세션 중 발견한 패턴, 베스트 프랙티스, 주의사항을 지식 베이스에 축적
- 반복되는 작업 패턴을 skill로 자동화
- 프로젝트 컨벤션과 가이드라인을 지속적으로 개선

---

## 실행 순서

### 1. 세션 회고
현재 세션에서 수행한 작업들을 검토하고 다음을 식별:

| 항목 | 확인 내용 |
|------|----------|
| 새로운 패턴 | 기존에 문서화되지 않은 코드 패턴이나 기법 |
| 삽질/실수 | 시행착오를 겪은 부분 (다음에 피할 수 있도록) |
| 베스트 프랙티스 | 효과적이었던 접근 방식 |
| 주의사항 | 발견된 함정이나 제약사항 |
| 반복 작업 | 자동화할 수 있는 반복적인 작업 흐름 |

### 2. CLAUDE.md 검토
```bash
cat CLAUDE.md
```

현재 세션의 학습 내용이 CLAUDE.md에 추가/수정될 필요가 있는지 확인:
- 새로운 컨벤션이나 규칙
- 기존 가이드라인의 보완/수정
- 새로운 주의사항 추가
- 코드 예시 업데이트

### 3. Skills(Commands) 검토
```bash
ls -la ~/.claude/commands/
```

기존 skills 수정 또는 새로운 skill 생성 필요 여부 확인:
- 기존 skill의 개선점
- 새로 자동화할 워크플로우
- 사용하지 않는 skill 정리

### 4. 사용자와 논의
식별된 개선사항을 사용자에게 제안:

```markdown
## 세션 리뷰 결과

### 학습한 내용
- ...

### CLAUDE.md 개선 제안
| 섹션 | 변경 내용 |
|------|----------|
| ... | ... |

### Skills 개선 제안
| Skill | 변경 내용 |
|-------|----------|
| ... | ... |

반영할까요?
```

### 5. 승인된 개선사항 반영
사용자 승인 후:
- CLAUDE.md 수정 (Edit 도구 사용)
- Skills 파일 생성/수정 (Write 도구 사용)
- 변경사항 요약 출력

---

## 체크리스트

### CLAUDE.md 검토 포인트
- [ ] 프로젝트 개요 - 새로운 기술 스택 추가?
- [ ] 주요 명령어 - 새로운 명령어 발견?
- [ ] 아키텍처 - 새로운 패턴이나 구조?
- [ ] 데이터 레이어 - DB/캐시 관련 새로운 지식?
- [ ] 에러 핸들링 - 새로운 에러 패턴?
- [ ] 코드 컨벤션 - 새로운 규칙이나 라이브러리?
- [ ] 주의사항 - 새로운 함정이나 제약?

### Skill 생성 기준
- 3회 이상 반복된 작업 패턴
- 여러 단계로 구성된 복잡한 워크플로우
- 실수하기 쉬운 작업 (자동화로 방지)

---

## 출력 형식

```markdown
## 세션 리뷰 완료

### 이번 세션 요약
- 주요 작업: ...
- 해결한 문제: ...

### 반영된 개선사항
| 대상 | 내용 |
|------|------|
| CLAUDE.md | ... |
| Skills | ... |

### 다음 세션을 위한 메모
- ...
```
