# 이미지 분석 스킬

이미지 파일을 읽고 분석하여 UI 상태, 에러, 문제 상황을 파악합니다.

## 입력 정보

$ARGUMENTS

## 작업 지침

### 1단계: 입력 파싱

사용자 입력에서 다음 정보를 추출하세요:
- `image_paths`: 이미지 파일 경로 (필수, 공백 구분으로 여러 개 가능, glob 패턴 지원)
- `context`: 추가 컨텍스트 정보 (선택, `context="..."` 형태)

### 2단계: 대상 이미지 파일 확인

1. **경로 처리**:
   - 상대 경로인 경우 현재 작업 디렉토리 기준으로 절대 경로로 변환
   - glob 패턴이 포함된 경우 Glob 도구로 매칭되는 파일 목록 확보
   - 지원 포맷: PNG, JPG/JPEG, GIF, WebP, BMP, TIFF 등 (Read 도구가 지원하는 모든 이미지)

2. **파일 존재 확인**:
   - Glob 도구로 대상 파일 존재 여부 확인
   - 파일이 없으면 에러 메시지 출력 후 종료

### 3단계: 이미지 분석

1. **이미지 읽기**:
   - Read 도구를 사용하여 모든 이미지 파일 읽기
   - **가능한 한 병렬로** 여러 이미지를 동시에 읽기 (Read 도구 병렬 호출)

2. **각 이미지 분석**:
   - UI 요소 식별 (버튼, 입력 필드, 네비게이션, 모달 등)
   - 화면에 표시된 텍스트 읽기 (에러 메시지, 알림, 라벨 등)
   - 에러 상태 감지 (404, 500, 빈 화면, 로딩 실패 등)
   - 레이아웃/스타일 문제 감지 (깨진 UI, 정렬 오류, 오버플로우 등)
   - 데이터 표시 상태 확인 (빈 목록, 잘못된 값, 누락된 정보 등)

3. **여러 이미지인 경우 비교 분석**:
   - 이미지 간 차이점 파악
   - 시퀀스로 보이는 경우 흐름 추적
   - 정상/비정상 상태 비교

### 4단계: 분석 결과 보고

다음 형식으로 결과를 보고하세요:

```
## 이미지 분석 결과

### 분석 대상
- 파일: <파일 목록>
- 총 이미지 수: <N>개

### 이미지별 분석

#### <파일명 1>
- **화면 유형**: (메인 화면 / 설정 / 에러 페이지 / 모달 등)
- **주요 UI 요소**: ...
- **표시된 텍스트**: ...
- **상태**: (정상 / 에러 / 로딩 중 등)

#### <파일명 2> (여러 이미지인 경우)
...

### 감지된 문제 (있는 경우)
- **문제 유형**: (에러 화면 / UI 버그 / 데이터 오류 등)
- **상세 설명**: ...
- **추정 원인**: ...

### 관련 정보 (있는 경우)
- 에러 메시지: ...
- 관련 URL/경로: ...
- 관련 데이터: ...
```

## 입력 예시

```
/path/to/screenshot.png
./screenshots/*.png
~/Downloads/error1.png ~/Downloads/error2.png
./bug.png context="결제 화면에서 금액이 0원으로 표시됨"
./screens/*.jpg context="회원가입 플로우 검토"
```

## 주의사항

- Read 도구는 이미지를 직접 읽을 수 있으므로 별도 변환 불필요
- 이미지가 많은 경우 병렬 읽기로 효율적으로 처리
- context가 주어진 경우 해당 맥락에 집중하여 분석
- 분석 결과는 비개발자도 이해할 수 있는 수준으로 작성
